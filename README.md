# Okawari Finder

`okawari-finder.user.js` は、Ingress の Orion 系おかわりCF計画を IITC 上で作るための userscript Plugin です。

15本の背骨ポータルを先に指定し、その背骨に対して固定の基点Aと、複数のおかわり基点Bを探します。Bはポータル密度が高い場所から優先して選び、AはそのB群と背骨15本で三角形が成立する位置から選びます。

## できること

- 背骨対象ポータル15本を指定する
- 現在画面内に読み込まれているポータルから、基点Aとおかわり基点Bを探索する
- 候補一覧を表示する
- 選択した候補を地図上に描画する
- Bを手動で追加する
- 計画プレビューを表示する
- 計画を JSON / Bookmark / DrawData / CSV で出力する
- 独自形式JSONから計画を読み込む

## 最初にすること

まず、背骨ポータルを15本選びます。

背骨とは、おかわりCFの中心になる一列のポータルです。Plugin上では `S1` から `S15` として扱います。

背骨対象ポータルの指定方法は2つあります。

1. `ブックマークから読込`
   - IITC Bookmark Plugin 等に登録されているポータルを読み込みます。
   - 読み込まれたポータル一覧から、背骨に使う15本を残します。
   - 不要なポータルは `削除` します。

2. `マップクリック追加`
   - 初期状態は `OFF` です。
   - `ON` にしてからマップ上のポータルをクリックすると、背骨対象ポータルに追加されます。
   - 15本そろったら `OFF` に戻すと誤追加を防げます。

背骨対象ポータルが15本ちょうどになったら、次に `計画生成` を押します。

## 計画生成

`計画生成` を押すと、現在画面内に読み込まれているポータルから A/B 候補を探索します。

背骨に指定した15本は探索対象から除外されます。AとBは、画面内ポータルから探します。

## Bの選び方

Bは「おかわりポータル」です。

Pluginはまず、画面内ポータルの中からB候補を作ります。B候補には以下の条件があります。

- 背骨15本すべてにリンクできる位置関係である
- Bから背骨へ引くリンクが、他の背骨ポータルに近すぎない
- 背骨列からある程度離れている

そのうえで、B候補の周辺密度を見ます。

`B密度半径m` の範囲内にB候補が多い場所ほど、おかわりしやすい場所として高く評価します。初期値は `400m` です。

たとえば `おかわりB` が `10` の場合、密度が高いクラスタから近いB候補を最大10件選びます。Bが密集しているほど、現地で複数のおかわり基点を使いやすい、という考え方です。

## Aの選び方

Aは固定基点です。

Pluginは、Bクラスタごとに最適なAを探します。A候補には以下の条件があります。

- A自身が背骨15本すべてにリンクできる
- AとBが背骨列の反対側にある
- A-Bを結んだ線と背骨15本で三角形が成立する
- A-Bリンクが背骨ポータルに被りすぎない
- すべてのBについて、A-B-背骨の構造が成立する

条件を満たすA候補の中から、形状スコアが高いものを選びます。

つまり、Bは「おかわりしやすい密度」を重視して先に探し、そのB群を成立させられるAを後から選ぶ、という流れです。

## 候補一覧

計画生成後、候補一覧が表示されます。

表示される主な項目は以下です。

- `Rank`: 候補順位
- `基点A`: その候補で使うAポータル
- `Shape`: 形状スコア
- `選択`: 候補を選択するボタン

候補は主にチーム総スコアを優先し、その次にB数、形状、おかわりしやすさなどで並びます。

## Bを手動で追加する

候補を選択すると、`おかわりCF計画` の中に `B手動追加` が表示されます。

`B手動追加: ON` にしてからマップ上のポータルをクリックすると、そのポータルを追加Bとしてチェックします。

追加Bは無制限に追加できます。

追加時には以下をチェックします。

- Aと背骨列の反対側にある
- Bから背骨15本すべてへリンクできる
- A-Bと背骨15本で三角形が成立する
- 被り判定にかからない
- A、背骨、既存Bと重複していない

OKなら計画に追加され、スコア、リンク、描画、出力内容にも反映されます。NGの場合は理由が表示され、追加されません。

## 地図に描画

`地図に描画` を押すと、選択中の計画を IITC の地図上に描画します。

描画されるものは以下です。

- 背骨 `S1` から `S15`
- 基点A
- 基点B群
- A/Bから背骨へのリンク予定線
- A-Bリンク
- 担当色分け

再描画時は、前回の描画レイヤーをクリアしてから描画します。

## 計画プレビュー

`計画プレビュー` を押すと、実行順の確認用ウィンドウが開きます。

プレビューでは、リンク手順を順番に確認できます。チェックボックスで進捗管理もできます。

担当A/B/Cごとの表示フィルタもあります。

出力画面から `計画プレビューCSV` を保存することもできます。

## 設定

`設定` ボタンを押すと、担当者名の入力欄が表示されます。

- `担当A`
- `担当B`
- `担当C`

具体的なエージェント名を入力できます。入力後、もう一度 `設定` を押すと反映されます。

通常表示では入力欄を隠しているので、スマホ画面でも幅を取りにくくしています。

その他の設定は以下です。

- `おかわりB`: 自動探索で何個のBを取るか。初期値は `10`
- `B密度半径m`: B密度を評価する半径。初期値は `400`
- `被り判定m`: リンクが背骨ポータルに近すぎるかを見る距離。初期値は `1`
- `マップ描画`: 候補選択時に地図へ描画するか

## 出力と保存

`出力` を押すと、別ウィンドウで保存用データを表示します。

出力形式は以下です。

- `JSON`
  - Okawari Finder 独自形式
  - 後から `計画読込` で読み込めます

- `Bookmark`
  - 使用するポータルのブックマークデータ
  - 背骨、A、Bを含みます

- `DrawData`
  - 背骨とA/Bからのリンク線を含むドローデータ

- `計画プレビューCSV`
  - 実行手順の確認用CSV

各形式はボタンで保存できるほか、下部の折りたたみ欄から内容をコピーできます。

## 計画読込

`計画読込` では、Okawari Finder 独自形式JSONを読み込めます。

読み込み時、既存の結果や背骨選択がある場合は、消してよいか確認します。

## 注意点

このPluginは、初版方針として位置関係だけを見て計画を作ります。

以下は考慮していません。

- 既存リンク
- 敵リンク
- Machinaリンク
- キー所持数
- 破壊可否
- SBUL
- 現地状況
- 実際の到達性

実行前に、現地状況と実リンク可否は必ず確認してください。
